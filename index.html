SOCIAL MEDIA API Documentation

The social media REST Api contains GET, POST, PATCH, DELETE requests but if you don't sign in it just allows you to GET requests.

Posts

The Posts resource represents all shared posts via registered users.

GET:
/posts

Returns a list of all current posts

GET:
/posts/:postId

Returns a single post record belonging to postId.

POST:
/posts

Used to create a new post .To send this request, you must be logged in and send it with the token in the header of the request.

PATCH:
/posts/:postId

This request will add comments or edit a post depending on the body fields of the request what you send.

DELETE:
/posts/:postId

Deletes the post belonging the postId.

Login

The Login routes includes registration and login proccess.

POST:
/signup

Submitted to create a user record. When the record is success, a new user record is created

POST:
/signin

Used to login.

Users

The Users resource represents all current users via registered users.

GET:
/users

Returns a list of all current registered users.

GET:
/users/search?q={username}

Returns users matching the parameter sent by query.

GET:
/users/:username

Returns the user belonging to username sent as parameter.

PATCH:
/users/:userId

Updates the user according to the body fields of the sent request.

DELETE:
/users/:userId

Deletes the user belonging to the userId sent as parameter.

Messages

The Messages resource represents all shared messages.

GET:
/messages

Used to get all messages.

GET:
/messages/:username

Returns all messages belonging to the user.

POST:
/messages

Used to create a new chat.

PATCH:
/messages/:messageId

It is used to add a new message to the chat belonging to messageId given as parameter..

Organizations

The Organizations resource represents all shared organizations.

GET:
/organizations

Used to get all organizations.

GET:
/organizations/:userId

Returns all organizations where the user of the userId given as parameter is registered.

PATCH:
/organizations/:orgId

It is used to add participants and announcements to the organization belonging to the orgId given as parameter.

POST:
/organizations

Used to create a new organization.
